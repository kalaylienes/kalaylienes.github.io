    IARVR 08 - AI Usage - TUDa 25/26

    Kalaylienes
    March 15, 2026

    During the development of this project, artificial intelligence tools were used as a supportive resource to assist with implementation and development tasks. Throughout the process, I used the browser-based ChatGPT 5.2 Thinking model through ChatGPT Plus Business as the primary AI tool.

    AI was mainly used to support the translation of ideas into code and to assist with repetitive programming tasks. However, the overall design, structure, and development process of the project were planned and implemented manually.

    In particular, I intentionally avoided using automated development workflows such as OpenAI Codex, OpenAI CLI-based agents, or tools like Claude Code, which can generate and modify larger parts of a project automatically. Instead, I chose to work with the browser-based interface so that each step of the implementation could be reviewed and applied manually.

    This approach ensured that the development process remained fully under my control. While it meant that the project sometimes progressed more slowly than it might have with fully automated workflows, it also allowed me to clearly understand and implement each system step by step.

    AI assistance proved especially useful when writing repetitive code structures or implementing supporting components that did not involve complex design decisions. In these cases, it significantly reduced development time.

    At the same time, the project also highlighted some limitations of current AI tools in software development. Although AI systems have made significant progress in recent years, they can still struggle with more complex systems when used without careful guidance. Because these models are trained on large collections of internet data, including outdated documentation and inconsistent forum discussions, some generated suggestions appeared plausible but overlooked small technical details.

    During development there were several cases where minor implementation issues or mismatches between Unity components caused problems that AI-generated suggestions did not fully resolve. In many of these situations, the final solution was reached through manual debugging and experimentation rather than through AI-generated guidance.

    Based on this experience, AI proved to be particularly effective as a tool for assisting with implementation and code generation, while complex system debugging still required careful manual investigation.

    Nevertheless, using AI provided a substantial productivity benefit and helped accelerate parts of the development process.

    For reference, the full Unity project is published here: Carrot-on-a-stick_IARVR2526.
